Why AIQA?

Because AI systems need more testing, not less. And testing should be easy.

Quality & Reliability

AI systems are unpredictable. They need more testing than traditional software, not less. AIQA helps you catch issues before they become problems. Because there's a reason pilots have licenses - shouldn't your AI systems have the same?

Ease of Use

No need to become a testing expert. AIQA integrates seamlessly with your existing workflow. Just add a decorator and you're tracing. Simple as that. No PhD in making things look bad required.

Built for AI

Traditional testing assumes deterministic outputs. AI doesn't work that way. AIQA understands probabilistic systems, handles subjective outputs, and helps you test what actually matters.

The AI Testing Problem

Most AI systems receive less testing than traditional software, even though they need more. It's an unfortunate situation: They need more testing because their outputs are unpredictable. But they're also harder to test because they're unpredictable, so they get less.

Many teams follow a "poke and hope" approach - tweaking prompts until something looks good. But what looks good in a demo might not work in the real world. And real-world mistakes can have real consequences.

Simple Example - Works in Your Language - 5 Minutes to Get Started (try it!) 
from aiqa import WithTracing, flush_tracing

# Just add a decorator
@WithTracing
def my_ai_function(prompt: str):
    # Your AI code here
    return ai_model.generate(prompt)

# That's it! Now you're tracing. 
# If you need to send traces mannually, e.g. in a short-lived script, call:
await flush_tracing()
import { withTracing } from '@aiqa/client';

// Just wrap your function
const myAiFunction = withTracing(async (prompt: string) => {
  // Your AI code here
  return await aiModel.generate(prompt);
}, { name: 'myAiFunction' });

// That's it! Now you're tracing.
import com.aiqa.client.AiqaClient;
import com.aiqa.client.WithTracing;

// Initialize once
AiqaClient.init();

// Use annotation
@WithTracing
public String myAiFunction(String prompt) {
    // Your AI code here
    return aiModel.generate(prompt);
}

// That's it! Now you're tracing.
import "github.com/winterwell/aiqa-client-go"

// Initialize once
aiqa.InitTracing("", "")

// Wrap your function
myAiFunction := aiqa.WithTracing(func(prompt string) string {
    // Your AI code here
    return aiModel.Generate(prompt)
})

// That's it! Now you're tracing.
# Send spans directly via REST API
curl -X POST https://your-aiqa-server.com/api/span \\
  -H "Content-Type: application/json" \\
  -H "X-API-Key: your-api-key" \\
  -d '{
    "name": "my_ai_function",
    "input": {"prompt": "Hello, AI!"},
    "output": {"result": "Generated response"},
    "attributes": {
      "component": "my-service"
    }
  }'

# Or use OpenTelemetry OTLP endpoint
curl -X POST https://your-aiqa-server.com/v1/traces \\
  -H "Content-Type: application/x-protobuf" \\
  -H "X-API-Key: your-api-key" \\
  --data-binary @trace.pb
